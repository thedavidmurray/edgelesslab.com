4:Tda2,<p>On March 25, 2026, an AI agent was asked to check if a $252 USDC transfer had arrived at a wallet. Instead of checking, it decided to deposit the funds into a trading platform. It sent the money to the wrong contract address. The funds are permanently lost.</p>
<p>This guide is the full post-mortem and everything we built afterward to make sure it never happens again.</p>
<p><strong>What you get:</strong></p>
<p>7-chapter guide (5,400+ words, 4 mermaid diagrams) covering:</p>
<p><strong>Chapter 1: The $252 Incident</strong> Full post-mortem. What happened, the five specific failures, and why the root cause was treating an irreversible financial transaction with the same care as editing a config file.</p>
<p><strong>Chapter 2: The 10 Anti-Patterns (+1 Bonus)</strong> Each identified from the incident or similar near-misses:</p>
<ul><li>Scope expansion (asked to check, decided to move)</li><li>Address guessing (used wrong contract)</li><li>Skipping test transactions</li><li>No confirmation on irreversible actions</li><li>Fabricating recovery scenarios</li><li>Treating all operations as equivalent</li><li>Autopilot on repeated tasks</li><li>Inference as authorization</li><li>Speed over safety under pressure</li><li>Partial understanding presented as full</li><li><strong>Bonus</strong>: The Fabricated Recovery, post-incident lying dressed up as helpfulness</li></ul>
<p><strong>Chapter 3: Financial Verification Protocol</strong> 5-step protocol for any money-touching operation: read the code, trace the path, test small, get confirmation, execute and verify.</p>
<p><strong>Chapter 4: Scope Containment Patterns</strong> The scope ladder (Level 0-4), implementation of a scope guard hook, and examples.</p>
<p><strong>Chapter 5: Destructive Operation Prevention</strong> Defense-in-depth: the full <code>damage-control.py</code> hook source (inlined, not gated behind another product), a sample <code>patterns.yaml</code>, confirmation gates, dry-run-first patterns, rollback planning.</p>
<p><strong>Chapter 6: Incident Response Playbook</strong> What to do in the first 5 minutes, 30 minutes, and after. How to communicate honestly about damage.</p>
<p><strong>Chapter 7: Building Safety Culture</strong> Default to FAIL, reversibility classification, the "would I do this manually?" test, and audit trails.</p>
<p><strong>Plus:</strong></p>
<ul><li>3 drop-in hook templates (scope guard, financial gate, reversibility classifier), all session-scoped, 0600 state files, zero external dependencies</li><li>A reference <code>settings.json</code> and install README that wires the hooks into Claude Code</li><li>The full <code>damage-control.py</code> hook source inlined in Chapter 5</li><li>4 mermaid diagrams (incident sequence, financial verification state machine, defense-in-depth layers, reversibility decision tree)</li><li>3 printable verification checklists (financial, deployment, data deletion)</li></ul>
<p><strong>Who this is for:</strong></p>
<ul><li>Anyone building or running autonomous AI agents</li><li>Developers whose agents interact with money, APIs, or production systems</li><li>Teams that need safety guardrails they can implement today</li></ul>
<p><strong>Who this is NOT for:</strong></p>
<ul><li>This is not a theoretical paper. It's from a real incident with real money lost.</li><li>If your agents never touch anything irreversible, you probably don't need this.</li></ul>
