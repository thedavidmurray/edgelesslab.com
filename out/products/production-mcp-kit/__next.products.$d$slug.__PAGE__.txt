4:Tf7a,<p>Every MCP tutorial shows you how to define a tool and return a result. None of them show you what breaks when two sessions hit the same endpoint, your API key leaks through error messages, or the server crashes at 3am with nobody watching.</p>
<p>This kit fills that gap. It comes from running 4+ MCP servers in production: a semantic search engine (Effect-TS + ChromaDB + SQLite fallback), an inter-session message router (SQLite + durable message queues), and utility servers that have survived months of daily use by autonomous agents.</p>
<p><strong>What you get:</strong></p>
<p><strong>3 production server templates:</strong></p>
<ol><li><strong>Typed MCP Server (Effect-TS)</strong>, schema-validated inputs, tagged errors via <code>Data.TaggedError</code>, dependency injection via Effect Layers, and a real SQLite-backed key-value store. Not a stub: a small but real implementation you can extend.</li></ol>
<ol><li><strong>API Proxy Server</strong>, wraps external APIs with auth (key from env, never from prompts), in-memory rate limiting, response caching with TTL, and error sanitization that catches six classes of leaked secret (OpenAI, Bearer, api_key, GitHub PAT, AWS, JWT).</li></ol>
<ol><li><strong>Stateful Hub Server</strong>, Bun + <code>bun:sqlite</code> (WAL mode) durable message router. Lease/ack lifecycle, dead-letter queue, background sweeper, HTTP <code>/health</code> and <code>/metrics</code> endpoints on port 9800. The same architecture used in production by Agent Bus and similar inter-session message routers.</li></ol>
<p><strong>Plus, as real, runnable files, not snippets:</strong></p>
<ul><li><strong><code>SubprocessBridge</code> (TypeScript)</strong>, long-lived child process, JSON-RPC over stdin/stdout, per-call timeout, exit handler, backpressure. Pairs with the Python bridge already in <code>scripts/bridge.py</code>.</li><li><strong><code>fallback.ts</code></strong>, ChromaDB to SQLite FTS5 to in-memory TF-IDF cascade with health-aware routing and exponential backoff. Real TypeScript file, not a guide snippet. Ships with passing tests.</li><li><strong>Docker deployment</strong>, multi-stage Dockerfile, Compose config, health checks against the actual <code>/health</code> endpoint in the stateful-hub template, persistent volume, non-root user.</li><li><strong>17 passing tests</strong>, 13 in <code>api-proxy</code> (sanitizer + fallback cascade), 4 in <code>typed-server</code> (tagged errors).</li><li><strong>5,500+ word guide</strong>, 11 chapters and 2 appendices, including three real production incident debriefs, an observability chapter (Prometheus shape), graceful shutdown recipes, and four mermaid architecture diagrams.</li></ul>
<p><strong>Production patterns included:</strong></p>
<ul><li>Auth middleware: API keys from environment, never exposed to MCP protocol</li><li>Rate limiting: per-tool, per-minute, with configurable thresholds</li><li>Response caching: TTL-based with automatic eviction</li><li>Error sanitization: 6 regex patterns catching OpenAI keys, Bearer tokens, api_key=, GitHub PATs, AWS access keys, and JWTs</li><li>SQLite: WAL mode, prepared statements, busy timeout, lease/ack lifecycle, dead-letter queue</li><li>Health endpoints: status, pending/leased/dead-letter counts, uptime, Prometheus <code>/metrics</code></li><li>Graceful shutdown: SIGTERM handler, in-flight drain, WAL checkpoint, bridge teardown</li></ul>
<p><strong>Who this is for:</strong></p>
<ul><li>Developers building MCP servers that need to run reliably</li><li>Teams deploying MCP servers to production environments</li><li>Anyone who's outgrown the basic MCP tutorial examples</li></ul>
<p><strong>Who this is NOT for:</strong></p>
<ul><li>Beginners still learning what MCP is (start with the MCP Starter Kit at $24)</li><li>Projects that only need simple, single-use tools</li></ul>
