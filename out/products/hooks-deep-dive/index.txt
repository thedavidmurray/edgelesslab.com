1d:Td54,<p>Most Claude Code hooks tutorials stop at "run a linter before commit." That's fine for a side project. It falls apart when you have agents making real decisions.</p>
<p>These hooks come from a production system that coordinates 5 autonomous AI agents across a Mac and a Hetzner VPS. One of those agents lost $252 of real money before the right hooks existed. Now it can't.</p>
<p><strong>What you get:</strong></p>
<p>15-chapter guide (~4,700 words) with 4 mermaid diagrams and the $252 postmortem sidebar, covering:</p>
<p><strong>Safety hooks:</strong></p>
<ul><li>damage-control.py, blocks destructive commands in real-time (rm -rf, force push, DROP TABLE). Custom YAML parser, no external dependencies, fires on every tool call</li><li>verify-completion.py, evidence-based task verification. Agents must prove they're done, not just claim it. Default-to-FAIL philosophy</li></ul>
<p><strong>Observability hooks:</strong></p>
<ul><li>post-tool-tracker.py, logs every tool invocation to SQLite with secrets scrubbing. Your flight recorder</li><li>trading-observer.py, multi-agent financial tracking with risk levels and alerting</li><li>sandbox-observer.py, code execution monitoring with agent correlation</li></ul>
<p><strong>Automation hooks:</strong></p>
<ul><li>skill-activation.py, progressive skill disclosure based on prompt content (SHA256 config integrity)</li><li>inbox-auto-claim.py, multi-session dispatch with two-phase inbox claiming</li><li>loop-stop-hook.py, prevents accidental exit during iterative loops</li></ul>
<p><strong>Preservation hooks:</strong></p>
<ul><li>pre-compact.py, backs up critical context before compaction. Secrets scrubbed, FIFO rotation</li><li>webfetch-archive.py, archives web fetches to Obsidian with YAML frontmatter</li><li>session-end.py, forced retrospectives on significant sessions. Blocks exit until learning is captured</li></ul>
<p><strong>Plus:</strong></p>
<ul><li>Hook composition patterns (chaining, conditional, environment-aware)</li><li>4 drop-in template hooks (pre-commit gate, cost tracker, notifications, secrets scanner)</li><li>Complete deployment guide with settings.json examples</li><li>patterns.yaml reference (25 dangerous command patterns, 3-tier path protection)</li><li>SQLite schema for the observability stack</li></ul>
<p><strong>Who this is for:</strong></p>
<ul><li>Developers using Claude Code who want more control over agent behavior</li><li>Teams running autonomous agents that need safety guardrails</li><li>Anyone who's been burned by an agent doing something destructive</li></ul>
<p><strong>Who this is NOT for:</strong></p>
<ul><li>Complete beginners to Claude Code (start with the free Memory Kit)</li><li>People who only use Claude Code for simple one-off tasks</li></ul>
<p><strong>Inside the package:</strong></p>
<ul><li>guide.md (15 chapters, ~4,700 words, 4 mermaid diagrams)</li><li>11 production hooks + 4 customizable templates (15 Python files total, each runnable standalone)</li><li>7 shared library modules under <code>content/lib/</code></li><li>3 config files (patterns.yaml, schema.sql, settings-example.json)</li><li>README with copy-paste install walkthrough and working test commands</li><li>CHANGELOG documenting known limitations and planned v1.1 items</li></ul>
<p>All code is Python 3.10+ with zero external dependencies.
