4:Tf29,<p>Most generative art tutorials give you a script that makes pretty patterns on screen. They don't tell you how to make those patterns work on a pen plotter, where stroke ordering matters, ink coverage needs to be in a specific range, and tiny details get lost in the mechanical precision of the pen.</p>
<p>This kit comes from 105+ experiments running an autoresearch loop across 75+ generators, scoring thousands of outputs to find what actually plots well.</p>
<p><strong>What you get:</strong></p>
<p><strong>10 working generators</strong> (all ship as standalone Python files, all share a common <code>plotter_utils.py</code> module):</p>
<ol><li>Flow Field, Perlin noise particle traces, stdlib only</li><li>L-System Tree, 5 presets (oak, willow, fern, bush, sierpinski)</li><li>Voronoi Cells, Lloyd's relaxation (requires numpy + scipy)</li><li>Spiral Patterns, Archimedean, logarithmic, hypotrochoid</li><li>Reaction-Diffusion, Gray-Scott (spots, worms, coral, maze, zebra presets, requires numpy)</li><li>Subdivision Grid, Vera Molnar-inspired recursive cuts</li><li>Subject-Driven Hatching, NO background hatching, bring your own image or use the built-in mask</li><li>Space-Filling Curves, Hilbert, Heighway dragon, Gosper flowsnake</li><li>Topographic Contours, multi-octave noise with ridged-multifractal mode (requires numpy)</li><li>Cellular Automata, 1-D elementary rules (30, 90, 110, 184, etc.)</li></ol>
<p><strong>30 pre-plotted example SVGs</strong> (three seeds per generator) in <code>content/examples/</code>, each with its seed, parameters and plot time recorded in a <code>MANIFEST.md</code> alongside PNG previews for every file.</p>
<p><strong>5,700-word plotter-focused guide</strong> (<code>content/guide.md</code>) with:</p>
<ul><li>Measured stroke-optimization before/after numbers (17-19x air-travel reduction on real output)</li><li>Four mermaid diagrams (scoring pipeline, stroke-optimization comparison, L-system expansion, generator-selection tree)</li><li>Failure-mode gallery (feed rate, wet ink, scoring rejects, re-plot cost)</li><li>30 named parameter recipes (three per generator, copy-paste CLI ready)</li><li>AxiDraw + iDraw setup walkthroughs with pen pressure, feed rate, paper pairing</li><li>An FAQ and a "common gotchas" section from nine months of autoresearch</li></ul>
<p><strong>Production features:</strong></p>
<ul><li>SVG output optimized for plotters (single-stroke, no fills)</li><li>Shared nearest-neighbor stroke ordering with measured 17-19x air-travel reduction</li><li>Built-in plot-time estimator (+/- 15% vs real wall clock)</li><li>Configurable paper sizes (A5, A4, A3, Letter) with margin-aware layout</li><li>Metadata embedded in every SVG (seed, full parameter dump, path count, draw travel, air travel, plot time, pen)</li><li>AI scoring script with Gemini and OpenAI providers, plus a real scoring log from a 10-piece gallery run (average 7.2 / 10)</li></ul>
<p><strong>Scoring rubric from 105+ experiments:</strong></p>
<ul><li>6 dimensions: composition, complexity, coherence, ink coverage, plottability, uniqueness</li><li>Quality gates: 8+ = print immediately, 6-7.9 = worth plotting, < 4 = rework</li><li>LLM scoring prompt included, run it on your outputs</li></ul>
<p><strong>Who this is for:</strong></p>
<ul><li>Developers with pen plotters (AxiDraw, iDraw, etc.) who want algorithmic art</li><li>Creative technologists exploring the intersection of code and physical art</li><li>Anyone curious about generative art with a focus on real output, not just screen renders</li></ul>
<p><strong>Who this is NOT for:</strong></p>
<ul><li>If you want Processing/p5.js sketches, this is Python-only</li><li>If you don't care about physical output, you won't use half the features</li></ul>
<p>All generators are Python 3.10+. Core generators have zero dependencies.
